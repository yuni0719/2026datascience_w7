# 🌸 暖暖日和 (Miss Chang Shop) - 跨平台同步開發案

這是一個專為「暖暖日和」品牌打造的數位轉型方案，包含 **RWD 響應式購物網頁**與**APK 應用程式**。

## 🎯 客戶需求分析與對策 (作業評分重點)
針對本案客戶張小姐（台北大安區）的特殊需求，本專案採取以下開發策略：
* **數位門檻對策**：客戶不擅長部落格與電子相簿操作。為降低維護難度，本專案捨棄複雜後台，直接採用 **GitHub 線上編輯** 作為同步管理介面，讓客戶能以最直覺的方式更新商品。
* **同步下載需求**：客戶希望手機桌面有專屬圖示，且網站與 APP 內容須 100% 同步。
* **商務溝通策略**：針對客戶以「AI 產出很快」或「贈送衣服」為由之降價要求，本專案堅持 **WebView 封裝技術** 與 **跨平台 RWD 適配** 的專業價值，確保內容一致性。

---

## 🚀 核心功能與實作技術

### 1. 響應式網頁 (RWD) 
* **視覺風格**：採日系溫暖粉色調（#FFD1DC）與米色系（#FFFDD0）。
* **技術架構**：HTML5 / CSS3 / Google Fonts / Material Icons。
* **線上預覽**：[https://yuni0719.github.io/2026datascience_w7/]

### 2. 同步 APK 檔案 
* **實作技術**：採用 **WebView 封裝技術**。
* **核心價值**：實現「一處修改，兩處更新」。APP 會即時讀取網站原始碼，解決重複維護的困擾。
* **下載位址**：請至本儲存庫右側 **[Releases]** 區塊下載 `.apk` 安裝檔。

### 3. 客戶同步管理後台 
* **管理網址**：`https://github.com/yuni0719/2026datascience_w7`
* **後台帳密**：為確保資安，實際交付採 **GitHub Collaborators** 模式。作業演示請參考 README 說明。
* **操作邏輯**：客戶登入後編輯 `index.html` 內的圖片路徑或售價，存檔後網站與 APK 將自動完成內容同步。
